Requirements using LiveBPMN.com

  1. Within a process, including embedded subprocesses, event-based subprocesses, and transaction subprocesses, one MUST NOT use the @ character in names of flow objects (activities, events and gateways).
  2. Within a process, including embedded subprocesses, event-based subprocesses, and transaction subprocesses, one MUST NOT use the BPMiNer character string in names of flow objects (activities, events and gateways).
  3. Communication in LiveBPMN.com relies on explicit message flows. Beyond, event emission/reception within a process or from a subprocess to its wrapping subprocess/process relies on the matching of event names. For instance, within a process, a link catch event is automatically triggered by a link throw event having the same case-sensitive name (or both have no name: not recommended).

Non-executable BPMN models

  1. deux flots (ou plus) de séquence sortant d'une activité sont du parallélisme implicite
  2. un processus a au moins un événement start et un événement end
  3. Etc.
  4. un flot de séquence ne franchit pas le contour d'un sous-processus
  5. tout événement intermédiaire a au moins un flot de séquence arrivant et un flot de séquence sortant
  6. une porte ne peut pas être à la fois fork et join
  7. une porte event-based a au moins deux flux de sortie dont les outgoing sont des tâches de réception de message ou des événements intermédiaires reçus parmi ?
